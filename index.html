1분면

빠른 마크업 결과물 전달

현재 문제: 설계 완료 후 마크업 작업에도 1달 소요되는 케이스 다수
해결 방안:
  • AI 마크업 생성 시 1주 내 완료 가능
  • CSS 컨벤션 문서 및 지UI 디자인 시스템 AI Friendly하게 다듬는 작업 진행 중
4분면

UX 기획팀 자체 개발 진행

현재 문제:
  • 용어 수정 등 간단한 개선도 의도적으로 개발팀에 전달해 왔음(기본적인 통일성 인식 향상 목적)
  • 개발만 후순위로 밀리고 일감 개수만 증가
  • 개발한 사람 따로 휴리스틱 일감 수정하는 따로. 정화 효과 반감
해결 방안: UX 기획팀 내에서 자체 일감 수정 처리(AI 적극 활용)
2분면

중요 UX 개선건 관리 대시보드 공유

현재 문제:
  • 전제품 전개 요구됨 - 팀별, 제품별 상이한 개발 로드맵
  • 사이트 요구 사항 대응 최우선, UX 개선 후순위
  • 하위 호환성 고려 등 복잡도가 높아 QA도 오래 걸림
해결 방안:
  • 관리 대시보드 공유 (현재상태, 완료예정일, 지체일)
  • 다바다허브에 포함하여 상시 조회 가능 + 매터모스트 전용 채널 주기적 알림
  • 매월 정기적 보고 항목으로 포함하여 모든 팀 및 임원진까지 공유

🔴 현재 문제

  • 개발은 되었으나 품질 테스트에서 실패, 개발팀과의 무한 핑퐁
  • 개발 일정 압박으로 제대로 된 테스트 미수행 → 품질 저하 및 릴리즈 지연

🔧 DLPCenter 전체 코드 리팩토링

  • UX 검토 강화 (현재도 수행 중)
  • 불필요한 추상화 레이어 제거
  • AI 토큰 감소 효과 + AI가 더 잘 이해할 수 있는 코드 베이스로 전환
  • 코딩 컨벤션 통일 (대소문자 혼재, value 유무 혼재 등 정리)
588개 백엔드 코드 삭제 가능 (59,604라인)

🧪 테스트 코드 작성 의무화

  • AI가 가장 잘하는 것: 테스트 코드 작성
  • 현재: 개발 일정 압박으로 테스트 코드 작성 거의 전무
해결 방안:
  • AI로 테스트 코드 작성
  • Sub Agent로 병렬 테스트 수행
  • Hook 기능으로 자동 테스트 실행

Issue 02

뒷북 개선에 따른 이중 비용 발생

🔴 현재 문제 흐름

⏭️ 급한 일정으로 UX 검토 SKIP
→
🐛 이상한 UI 적용
→
🔀 stable 버전 머지
→
🔍 휴리스틱 활동으로 뒤늦게 발견
→
💸 이중 비용 발생
  • 기본적인 용어, UI 틀어짐, 버튼 색상 및 아이콘 배치 등이 잘못된 상태로 머지
  • 개발자는 UX 검토를 번거로운 절차로 부정적 인식 가능
  • 확인된 이상 수정 불가피 → 개발팀에 일감 발행 → 후순위로 밀리는 악순환

🟢 개선된 흐름 (AI 자동화)

🏗️ QA 요청 시 젠킨스 브랜치 빌드
→
🤖 AI가 JSP/HTML 변경 여부 확인
→
📋 AI가 일감, 내용 요약, 영향도 정리
→
💬 매터모스트 UX팀 채널 즉각 알림
→
✅ 선제적 수정 및 머지

✨ 기대 효과

QA 요청과 동시에 UI 검토 수행 가능 → 수정 및 UI 검토 완료 상태로 stable 버전 머지 되어 안정성 향상
